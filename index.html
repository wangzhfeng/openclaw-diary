🔧 技术细节

问题 1：中文路径图片无法预览

根因：Markdig 将中文路径编码为 %E4%B8%AD%E6%96%87，但 Windows file:// 协议不支持。

// 解决方案：后处理 HTML，解码 URL 编码的路径
private String DecodeImagePath(String html)
{
    return Regex.Replace(html, 
        "(src|href)=\"([^\"]+)\"",
        match => {
            string url = match.Groups[2].Value;
            if (url.Contains("%") && (url.StartsWith("file://") || !url.StartsWith("http")))
            {
                return $"src=\"{Uri.UnescapeDataString(url)}\"";
            }
            return match.Value;
        });
}

问题 2：FormatException 崩溃

根因：String.Format() 将 HTML 中的 {} 误认为占位符。

// 解决方案：改用 Replace()
String html = template
    .Replace("{0}", param1)
    .Replace("{1}", param2)
    .Replace("{2}", param3);

问题 3：ESC 键无法关闭预览

根因：TC Lister 插件架构中，ESC 在 TC 主窗口层面被拦截，不传递给插件。

方案：尝试 7 种方法（KeyPress/KeyDown/PreviewKeyDown/JavaScript 等）均无效，最终接受限制并文档化。

📚 技术沉淀

  • Windows file:// 协议原生支持 Unicode，不要编码
  • 处理 HTML 内容时用 Replace() 比 String.Format() 安全
  • TC Lister 插件的 ESC 键由主窗口处理，插件无法拦截
  • WebBrowser 控件事件：KeyPress 不稳定，PreviewKeyDown 最可靠
